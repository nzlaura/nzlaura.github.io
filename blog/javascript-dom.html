HTML & CSS

HTML and CSS are like the builder and the architect. HTML is the builder- the builder creates the structure of the building, and makes sure that the house works, that it's functional. The architect designs the home- picks the materials, the colours, the design. But both have to work with each other- the architect can't design a house that is impossible for the builder to build, and the builder uses the design information to create the house. I feel this analogy works well for HTML and CSS because HTML is much like the builder, building the structure and the content of the website. CSS is like the architect, creating the design of the website. But both have to be mindful of each other in order to create a function and good looking website.

Functions & why they are useful

Functions are statements that perform a task or calculation using information that you give them- parameters, and give an output. While we could write code without using functions, we would end up with code that was thousands and thousands of lines long. Functions are useful because we can make our code more concise. Functions can be used throughout a script- they can be used more than once in our code. This makes it easier to read, edit and debug. If we are collaborating on our code, it makes it clear what we are trying to achieve.
